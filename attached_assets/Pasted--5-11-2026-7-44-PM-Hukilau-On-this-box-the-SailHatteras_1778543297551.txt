[5/11/2026 7:47 PM] Hukilau: Here’s a concrete curl you can run against this Gateway to send a single chat message and get a reply back, using the OpenAI-compatible endpoint.

Assumptions:

• Gateway is on this box at http://127.0.0.1:18789
• You have your Gateway token/password (replace YOUR_TOKEN_HERE)

curl -sS http://127.0.0.1:18789/v1/chat/completions \
  -H 'Authorization: Bearer YOUR_TOKEN_HERE' \
  -H 'Content-Type: application/json' \
  -d '{
    "model": "openclaw/default",
    "messages": [
      {
        "role": "user",
        "content": "Hello Hukilau, this is a test from Replit."
      }
    ]
  }'

Key pieces:

• Endpoint path: /v1/chat/completions
• Full URL: http://127.0.0.1:18789/v1/chat/completions
• Headers:
  • Authorization: Bearer YOUR_TOKEN_HERE
(use gateway.auth.token or gateway.auth.password, depending on your config)
  • Content-Type: application/json
• JSON body:
  • model: "openclaw/default" (routes to the main agent, i.e., me)
  • messages: standard OpenAI-style array with {role, content}

For Replit, you’d just swap 127.0.0.1 for whatever host:port you expose (or tunnel) the Gateway on, and keep the same path, headers, and body format.
